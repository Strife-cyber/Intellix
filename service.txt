╔══════════════════════════════════════════════════════════════╗
║  micro-cer — Docker Compose integration                     ║
║  Add the snippet below to the main project's                ║
║  docker-compose.yml under the `services:` section.          ║
╚══════════════════════════════════════════════════════════════╝

  micro-cer:
    build:
      context: ./micro-cer
      dockerfile: Dockerfile
    container_name: micro-cer
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      # Required: shared HMAC secret (must match the value in the frontend proxy)
      CER_SHARED_SECRET: "${MICRO_CER_SECRET:-change-me-shared-secret}"

      # AI provider configuration (choose one or more)
      AI_PROVIDER: "${MICRO_CER_AI_PROVIDER:-openai}"

      # OpenAI
      OPENAI_API_KEY: "${OPENAI_API_KEY:-}"
      OPENAI_MODEL: "${OPENAI_MODEL:-gpt-4o}"

      # Google Gemini
      GEMINI_API_KEY: "${GEMINI_API_KEY:-}"
      GEMINI_MODEL: "${GEMINI_MODEL:-}"

      # Ollama (local)
      OLLAMA_BASE_URL: "${OLLAMA_BASE_URL:-http://ollama:11434}"
      OLLAMA_MODEL: "${OLLAMA_MODEL:-llama3.1}"

      # LM Studio (local, OpenAI-compatible)
      LM_STUDIO_BASE_URL: "${LM_STUDIO_BASE_URL:-}"

      # OpenRouter
      OPENROUTER_API_KEY: "${OPENROUTER_API_KEY:-}"
      OPENROUTER_MODEL: "${OPENROUTER_MODEL:-}"
    volumes:
      # Persist user uploads, job outputs, and custom templates
      - micro-cer-data:/app/data
    networks:
      - default  # or your app's shared network name, e.g. intellix-network
    depends_on:
      # If using Ollama as the AI provider
      # - ollama
      # If using an S3-compatible store or database
      # - minio
      # - postgres
      {}

volumes:
  micro-cer-data:
    driver: local

╔══════════════════════════════════════════════════════════════╗
║  Required headers from the frontend proxy                   ║
║  Every /api/* request must include these HTTP headers:      ║
╚══════════════════════════════════════════════════════════════╝

  X-CER-USER-KEY    —  Unique user identifier (string)
  X-CER-TIMESTAMP   —  Unix epoch seconds (string)
  X-CER-SIGNATURE   —  HMAC-SHA256 hex digest

  Signature algorithm (pseudo-code):
      payload = userKey + "|" + timestamp + "|" + method + "|" + apiPath + "|" + SHA256(body)
      signature = HEX(HMAC-SHA256(payload, secret))

  For multipart/form-data, the body hash is SHA256("") (empty input).

╔══════════════════════════════════════════════════════════════╗
║  Directory reference inside the container                   ║
╚══════════════════════════════════════════════════════════════╝

  /app/micro-cer              —  Compiled binary
  /app/internal/template/     —  Built-in LaTeX templates
  /app/data/users/            —  Job outputs and prosits (persistent)
  /app/data/templates/        —  Custom uploaded templates (persistent)
  /app/uploads/               —  Transient upload scratch space
